## Step 3 で実装したもの

- [`firmware/src/proto.h`](../firmware/src/proto.h) / [`firmware/src/proto.cpp`](../firmware/src/proto.cpp)
  - LF 終端 ASCII 1 行を 256B バッファで組み立て → 2 文字大文字コマンド + payload に分解
  - `Parser(EmitFn)` インスタンスで `feed_byte(b)` するだけ。Pico SDK / Verilator 非依存の pure C++17
  - 実装済み: `PI` → `PO`、`VE` → `VE01reversi-fw`
  - 既知だが未実装 (`MO`/`PA`/`BO`/`SB`/`SW`/`EB`/`EW`/`ED`) は `ER02 not implemented`
  - 不正入力 (未知 / 小文字 / CR / 行長超過 / 非 ASCII) はそれぞれ `ER0n`
- [`firmware/src/main.cpp`](../firmware/src/main.cpp) を更新: `getchar_timeout_us(0)` で非ブロッキング受信 → parser に投入
- [`verif/tb_hil/test_proto.py`](../verif/tb_hil/test_proto.py) (HIL): 10/10 PASS (Probe UART 経由)

実装ノート:

- パーサが pure C++ なので、将来 cocotb 風の host 単体テスト (gtest 等) を追加する余地あり。今は HIL のみ
- 出力は `EmitFn` コールバック方式。main.cpp は `fwrite + fputc('\n')` で `stdio_usb` / `stdio_uart` どちらにも流す
- `etc/protocol.md §4` の CR 禁止は ER03 で守っている。WSL 側の `printf "PI\n"` は LF のみなので問題ない

  `verilated_trace.h` を **include guard 先取り定義** で取り込みごと潰し、
  必要な型 (`VlThreadPool`, `VerilatedTraceBaseC`, `VerilatedTraceConfig`) を最小 stub

## Step 4 着手時の前提

- `verif/tb_unit/` に cocotb の入れ物がある → `test_legal_bb.py` を足すだけ
- Python golden は [`reversi_rules.py`](https://github.com/open-tommie/24-mmo-Tommie-chat) (tommieChat 側)。
  HIL / cocotb のどちらからも import して `assert dut == reversi_rules.legal_moves(...)` で比較する設計
- 全 64 マス × Black / White × 任意盤面の **網羅は厳しいので、最初は典型局面 (initial / mid game / corner trap など)** で開始
- bit_index = `r*8 + c` (`a1 = bit0`)。エンディアンは Python と一致させる
- 手の選択順序は **行優先 a1, b1, c1, ..., h8** (旧プロジェクト失敗箇所)
